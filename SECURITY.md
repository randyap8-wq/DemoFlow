# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability in DemoFlow, please
**do not** open a public GitHub issue. Instead, report it privately by either:

- Opening a [GitHub Security Advisory](https://github.com/randyap8-wq/DemoFlow/security/advisories/new)
  on this repository, or
- Emailing the maintainers at the address listed on the project owner's GitHub
  profile.

Please include enough information for us to reproduce the issue (a minimal
proof-of-concept demo script, browser + version, and the affected DemoFlow
commit). We will acknowledge receipt within a reasonable time and keep you
updated as we investigate.

## Supported Versions

DemoFlow is pre-1.0; security fixes are made on the `main` branch. We do not
backport fixes to earlier tagged releases.
